# T03.5.1_JD-DATA030_Design

## Validation Strategy
- Flag missing critical fields as errors (e.g., total).
- Flag invalid numeric values as warnings.
- Detect subtotal/tax/total mismatches and line-item subtotal mismatch.
- Produce machine-readable `validation_errors`, `validation_warnings`, and `review_reasons`.

## Review Routing
- `review_required` is set when errors or warnings exist.
- `review_reasons` contains normalized, deduplicated reason codes.
