# T01.1.1 – Create requirements.txt with Python Dependencies

**Associated Requirement:** R01.1 (Dev Environment Reproducibility)

## Task Statement
Create a comprehensive `requirements.txt` file that specifies exact versions of all Python dependencies needed for the Local Document Extraction Copilot MVP. This file is the source of truth for environment reproducibility across all machines and CI/CD.

## Dependencies to Include

### Core Backend
- `fastapi==0.109.0` – HTTP API framework
- `uvicorn==0.27.0` – ASGI server
- `pydantic==2.5.0` – Data validation
- `pydantic-settings==2.1.0` – Config management

### Database & ORM
- `sqlalchemy==2.0.23` – ORM and SQL toolkit
- `psycopg2-binary==2.9.9` – PostgreSQL adapter
- `alembic==1.13.0` – Database migrations

### LLM & Vector Embeddings
- `ollama==0.1.33` – Ollama Python client (or equivalent latest)
- `sentence-transformers==2.2.2` – Embeddings generation
- `pgvector==0.2.0` – PostgreSQL vector extension

### Observability
- `opentelemetry-api==1.21.0` – OpenTelemetry API
- `opentelemetry-sdk==1.21.0` – OpenTelemetry SDK
- `opentelemetry-exporter-otlp==0.42b0` – OTLP exporter
- `opentelemetry-instrumentation-fastapi==0.42b0` – FastAPI instrumentation
- `opentelemetry-instrumentation-sqlalchemy==0.42b0` – SQLAlchemy instrumentation

### Utilities
- `python-dotenv==1.0.0` – Environment variable loading
- `python-multipart==0.0.6` – File upload handling
- `httpx==0.25.2` – HTTP client

### Development & Testing
- `pytest==7.4.3` – Testing framework
- `pytest-asyncio==0.21.1` – Async test support
- `pytest-cov==4.1.0` – Coverage reporting
- `black==23.12.0` – Code formatter
- `ruff==0.1.8` – Linter
- `mypy==1.7.1` – Type checker

## Acceptance Criteria
- [ ] `requirements.txt` exists at repository root
- [ ] All versions are pinned exactly (no `>=`, `~=`, etc.)
- [ ] File is separated into sections: Core, Database, LLM, Observability, Dev
- [ ] Each dependency includes a comment explaining its purpose
- [ ] `pip install -r requirements.txt` completes without errors
- [ ] Can verify all installed versions match requirements.txt
- [ ] No transitive dependency conflicts

## Definition of Done
- [ ] `requirements.txt` created and committed to repo
- [ ] `pip install -r requirements.txt` tested and passing
- [ ] Evidence: `pip list` output saved to `evidence/T01.1.1_requirements_list.txt`
- [ ] DEV-024 confirms acceptance
